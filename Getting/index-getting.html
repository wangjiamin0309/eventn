Step 2: Create A New Microservice

Creating a service will generate an internet facing REST resource with a unique URL, sandboxedPOSTandGETfunctions and a dedicated data store table.

User Interface

Login and select theCreate Servicebutton or plus icon:

API

Create a new service by sending aPOSTrequest tohttps://api.eventn.com/service

Authentication

A valid authentication token must be generated in order to use any of theapi.eventn.comresources. To generate a token, send aPOSTrequest tohttps://api.eventn.com/authenticationwith a body payload containingemailandpasswordkey values:

/Users/eventn/

  • curl -H "Content-Type: application/json" -X POST -d '{"email":"EMAIL","password":"PASS"}' https://api.eventn.com/authentication
  • {"status":"success","data":{"token":"my_token","user":{"id":2029,"first_name":"Event","last_name":"n","email":"hello@eventn.com"}}}

With the returned access token, set anAuthorizationheader and create the new service by making aPOSTrequest tohttps://api.eventn.com/service:

/Users/eventn/

  • curl -H "Content-Type: application/json" -X POST -d '{"email":"EMAIL","password":"PASS"}' https://api.eventn.com/authentication
  • {"status":"success","data":{"token":"my_token","user":{"id":2029,"first_name":"Event","last_name":"n","email":"hello@eventn.com"}}}

Take note of the returnedtokenfrom the service creation request as this is needed to access the service.
